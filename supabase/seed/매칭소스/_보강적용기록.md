# drugs 보강 적용 기록 — narcotic_type / prescription_type (2026 적재)

> 채팅 첨부 CSV는 UTF-8→Latin-1 mojibake 표시본이라 **수천 행 원문 재구성은 손상 위험**.
> 두 보강은 **규칙 기반**이므로 외부 한글 텍스트 의존 없이 DB의 정본 `drug_code`로 게이트 적용했다.
> 방식: owner 권한 가역 SQL(BEGIN/ROLLBACK 검증 → 본 적용). `narcotic_type/prescription_type='확인필요'` 행만 변경(가산적). 비밀값 노출 0.

## narcotic_type — 확인필요 225 → 0
- 규칙(보강_마약구분.csv·_README): 향정마약관리 목록=향정/마약, 그 외 확인필요=일반.
- 적용: `확인필요 → 일반` 일괄, 예외 `IRCD5 → 마약`(아이알코돈정 5mg).
- 결과 분포: 일반 224 · 마약 4(기존 3 + IRCD5) · 향정 13(불변) · **확인필요 0**.
- 롤백: `update drugs set narcotic_type='확인필요' where tenant_id=<cnc> and drug_code in (...224 codes...);`
  (역적용 필요 시 적용 직전 스냅샷/백업 기준 복원 권장)

## 미적재(차단) — 실데이터 파일 필요
