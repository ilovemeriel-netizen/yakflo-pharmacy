## Task2 — 보정안 (구입단가 기준 통일, 적용 전 승인 대기)
**원칙: 수량 절대 불변, 06월 금액만 재계산.** RPC·drugs·01~05월 무수정.

### 보정 대상 = monthly_snapshots WHERE snap_year=2026 AND snap_month=6 (금액 컬럼만)
- 약품별 **구입단가** = 최근 클린월(05→04→…→01) 같은 drug_code의 `closing_amount/closing_qty`(qty>0). 없으면 `drugs.edi_price`, 그것도 0이면 현 단가 유지(보류).
- 재계산: `opening_amount := opening_qty × 구입단가` · `closing_amount := closing_qty × 구입단가` · `subtotal_amount := subtotal_qty × 구입단가`. (06 in/out/disp/ret = 0 → 해당 금액 0 유지)
- 적용 경로: anon+RLS owner 세션 UPDATE(데이터). **BEGIN/ROLLBACK 가역 검증 후 본 적용**.
- **역방향 롤백**: 보정 전 06 원본 금액을 `_금액단가_06_롤백.sql`로 캡처(행별 UPDATE 복원). 스키마/RPC 변경 없음 → DDL 롤백 불요.
