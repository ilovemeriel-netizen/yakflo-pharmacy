# 월마감 2026-01~05 NOCODE 누락 정합 기록 (출처명 대조 + 정규화)

> 스크립트: `scripts/reconcile_monthly_nocode.mjs` (anon+RLS owner 세션, 미리보기→`--commit`)
> 원칙: 가산적·append-only(upsert만). 정규화·대조는 **비교 전용** — 원본 약품명/정본 값 변경 없음.
> 정책: **중복이 아닌 한 모든 기초데이터 등록**(누락 수용 안 함). 신규코드/등재 여부는 사용자 결정.

## 1) 매칭 단계 (우선순위)
① CSV drug_code → ② 약품명 정확일치 → ③ `norm1`(NFKC·공백·대소문자) → ④ `norm2`(기호 제거)
→ ⑤ `norm3`(**괄호 주석 제거**) → ⑥ `norm4`(**`/` 앞부분**). 다중 후보(모호)는 자동매칭 제외.
정규화는 비교 전용 — 원본/정본 무변경.

## 2) 회복 결과 (가산 upsert, 신규코드 0)
- **총 회복 +149행** (norm1/2 그대로 포함 후 norm3 +146 · norm4 +3). 전부 **기존 정본 코드**로 매칭(신규코드 없음).
- 핵심: 월마감명 뒤 변경/중단 주석 때문에 안 맞던 행을 괄호·`/` 제거로 회복.
  - 예: `디프렌캡슐20mg(로틴캡슐 변경예정)`→`디프렌캡슐20mg`(DPR20), `리큅정0.25mg(뉴큅정으로 변경)`→`리큅정0.25mg`(NOCODE-0021), `팔로델정/사용기한,사용량저조`→`팔로델정`(PLD2).
- 신규 (snap_month, drug_code) 키만 upsert. 재실행 시 회복 0행(멱등·가산 확인).

## 3) 잔여 10종 전부 등록 (정책: 판단 없이 모두 등록) — `scripts/register_monthly_residual.mjs`
사용자 지시("중복 아닌 한 모두 등록")에 따라 잔여 10종을 **드롭 없이** 등록:
- **모호 4종 → 기존 코드 결정적 배정**(동일 후보군 내 정렬 1:1, 신규코드 0):
  - 뮤테인캡슐200g(비엘아세틸캡슐 변경) → **NOCODE-0043**
  - 대한멸균생리식염수100ml/20개(병) → **D0432**
  - 스카이셀플루4가PFS 0.5ml_(무료독감) → **NOCODE-0256** · (지자체) → **NOCODE-0257**
  - (정본 동일명 복수는 의도된 데이터 — 공란은 추후 공공데이터 API 보완 예정)
