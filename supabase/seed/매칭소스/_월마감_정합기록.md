# 월마감 2026-01~05 NOCODE 누락 정합 기록 (출처명 대조 + 정규화)

> 스크립트: `scripts/reconcile_monthly_nocode.mjs` (anon+RLS owner 세션, 미리보기→`--commit`)
> 원칙: 가산적·append-only(upsert만). 정규화·대조는 **비교 전용** — 원본 약품명/정본 값 변경 없음.

## 1) 무엇을 했나
월마감(연간보고서) NOCODE 행의 약품명을 drugs 정본명(= NOCODE-#### 생성 출처 = yakflo_data명)과
대조해 drug_code를 복원. 매칭 우선순위: ① CSV drug_code → ② 약품명 정확일치 →
③ `norm1`(NFKC 전각·연속공백·대소문자 통일) → ④ `norm2`(괄호·중점·하이픈·`/`·공백 제거).
다중 매핑(모호)은 자동매칭에서 제외(이번 0건).

## 3) 완전성 표 (2026-06 = 1103 기준)
| 월 | CSV원천 | 적재(코드有) | 06대비격차 | 격차 분해: 06신설 / 중지보류 / 구조적중복 |
|---|---|---|---|---|
| 2026-01 | 1070 | **1026** | 77 | 33 / 41 / 3 |
| 2026-02 | 1080 | **1041** | 62 | 23 / 39 / 0 |
| 2026-03 | 1087 | **1048** | 55 | 16 / 39 / 0 |
| 2026-04 | 1092 | **1054** | 49 | 11 / 38 / 0 |
| 2026-05 | 1103 | **1066** | 37 | 0 / 37 / 0 |
| 2026-06 | 1103 | 1103 | 0 | (yakflo_data 완전 시드, 무영향) |

**격차는 데이터 손실이 아니다.** 각 CSV원천 행은 전부 회계된다:
`CSV원천 = 적재(코드有) + 중지보류 + 구조적중복`. (예: 01 → 1026+41+3 = 1070 ✓)
- **06신설**: 6월까지 정본에 추가된 약품 → 앞선 달 보고서엔 애초에 없음(정상).
- **중지보류**: 코드 없는 단종(전 행 0) → 범위 외, 적재 보류(아래 4).
- **구조적중복**: 서로 다른 약품명이 같은 코드로 귀결 → 유니크 인덱스상 1행만 존재(01의 3건).

- 결과: **사용 0 · 휴면 0 · 중지 41**. 전부 거래·재고 0인 단종 약품이며 이름에 *생산중단/변경/대체/폐기/코드중지* 표기.
