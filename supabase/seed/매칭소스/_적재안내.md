# 기초 데이터 적재 — 매칭소스 안내 (2026-01~05)

> 이 폴더에 **정식 UTF-8(BOM) CSV 5종**을 두면 `scripts/load_seed_2026.mjs`가 anon+RLS로 가역 적재한다.
> 채팅에 붙여 넣은 mojibake 표시본이 아니라, pandas `encoding='utf-8-sig'`로 열리는 원본 파일을 둔다.

## 둘 파일 (정확한 파일명)
| 파일 | 대상 | 행수 | 용도 |
|---|---|---|---|
| `약품_정본.csv` | drugs | 1103×31 | 대조(빈값/확인필요만 보강, 유효값 불변) |
| `보강_마약구분.csv` | drugs.narcotic_type | 225 | 확인필요 225→0 (224 일반 + IRCD5 마약) |

| `초기재고.csv` | inventory_stock | 1103 | 이월+0 대조(current_qty=opening_qty) |
| `월마감_2026.csv` | monthly_snapshots | 5432 | 2026-01~05 upsert |

## 스키마 매핑 (실측 — ALTER 불필요)
- `monthly_snapshots`: `snapshot_month "2026-01"` → **snap_year=2026 + snap_month=1** (snapshot_month 컬럼 없음, 분리 매핑).
  - opening_amt→opening_amount · in_qty→total_in_qty · in_amt→total_in_amount · used_qty→total_out_qty ·
    used_amt→total_out_amount · disposal_qty→total_disp_qty · return_qty→total_ret_qty · closing_amt→closing_amount.
  - subtotal_qty/amount: CSV 무 → opening+in 파생. code_status·약품명: 컬럼 없음 → 드롭.
  - upsert 유니크 키: (tenant_id, snap_year, snap_month, drug_code).
- `inventory_stock`: 이동 컬럼(in/used/disposal/return) **없음**. 이월+0 = `current_qty`(=opening_qty) 단일.
- NOCODE 행(빈 drug_code): DB drugs의 `drug_name → drug_code(NOCODE-####)` 맵으로 런타임 매칭(깨진 텍스트 비의존).

## 가역성 · 정리 범위
- drugs 보강: `narcotic_type='확인필요'` / `prescription_type='확인필요'` 행만 UPDATE (가산적). 롤백 = 해당 행 값을 '확인필요'로 되돌림.
- monthly_snapshots: 2026-01~05만 insert/upsert. 롤백/정리 = `delete ... where snap_year=2026 and snap_month in (1,2,3,4,5)` (식별 범위 한정).
- inventory_stock: **대조만**(유효 현재고 덮어쓰기 금지). 차이는 보고만.
- transactions: 초기 적재 시 0행 유지(어떤 행도 넣지 않음).

## 실행
```
# 미리보기(쓰기 안 함)
node scripts/load_seed_2026.mjs
# 본 적용(가역검증 통과 후)
node scripts/load_seed_2026.mjs --commit
```
환경변수(셸 한정, 커밋 금지): `SUPABASE_URL`, `SUPABASE_ANON_KEY`, 그리고 owner 로그인은 `.owner-login.local`(gitignored).
