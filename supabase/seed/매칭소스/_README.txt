· 조인키: drug_code. 코드 없는 약품은 code_status=NOCODE → 약품명으로 DB의 NOCODE-#### 합성코드에 매칭.
· 적재 원칙(가산적): DB가 빈값/확인필요/미시드인 곳만 채운다. 유효한 기존 값은 절대 덮어쓰지 않는다.
· 모든 쓰기는 가역 테스트(BEGIN/ROLLBACK 또는 테스트→검증→롤백) 후 본 적용.

--------------------------------------------------------------------
[파일 1] 약품_정본.csv  →  테이블: drugs   (1103행 × 31컬럼)
--------------------------------------------------------------------
마스터 속성 전체. Excel→DB 컬럼 매핑 (확정=스키마 확인됨 / 확인=information_schema로 검증):
  drug_code→drug_code(확정) · 약품명→drug_name(확정) · 성분명KR→ingredient_kr(확인) · 성분명EN→ingredient_en(확인)
  구분→category(확인) · 복합단일→compound_type(확정) · 전문일반→prescription_type(확정) · 마약구분→narcotic_type(확정)
  ATC번호→atc_code(확인) · 대/중/소분류→atc_l1/l2/l3(확인) · 효능→efficacy(확인) · 제조판매사→manufacturer(확인)
  총수량→total_qty(확인) · 제형→dosage_form(확인) · 포장→packaging(확인) · 첨가제→additives(확인)
  급여구분→reimbursement_type(확인) · 보험코드→insurance_code(확정) · 보험약가→edi_price(확정)
  통당단가→pack_price(확인) · 구입단가→purchase_price(확인)
  유효기한→expiry_date(확인) · 보관방법→storage_method(확정) · 보관위치→storage_location(확인)
  입고일자→received_date(확인) · 상태→status(확정) · 비고→remarks(확인)
  ※ 전월재고 컬럼도 포함되나, inventory_stock 시드는 [파일 2]를 정본으로 사용.

--------------------------------------------------------------------
[파일 2] 초기재고.csv  →  테이블: inventory_stock   (1103행)   ★이월+0 원칙★
--------------------------------------------------------------------
기초 재고 = 전월재고량(yakflo_data X열). 입고·사용·폐기·반품은 모두 0으로 세팅.
  opening_qty(전월재고) → 기준값
  in_qty=0 · used_qty=0 · disposal_qty=0 · return_qty=0   (모든 이동 0)
  current_qty = 전월재고 (= opening + 0 이동)  → inventory_stock.current_qty
· 즉 시스템 시작 상태: 현재고 = 전월재고, 어떤 이동도 기록되지 않음.
· DB가 이미 시드됐다면 대조·검증용(차이만 보고, 유효값 덮어쓰기 금지).

--------------------------------------------------------------------
[파일 3] 월마감_2026.csv  →  테이블: monthly_snapshots   (5432행 = 2026-01~05)
--------------------------------------------------------------------
출처: 연간보고서 2026-01월~05월. 월별 실제 월마감(전월재고→입고→사용→폐기→반품→현재고).
  snapshot_month(YYYY-MM) · drug_code · 약품명→drug_name
  opening_qty→opening_qty · in_qty→in_qty · used_qty→used_qty · disposal_qty→disposal_qty
  return_qty→return_qty · closing_qty→closing_qty · (opening_amt/in_amt/used_amt/closing_amt → *_amount)
· 유니크 키: (tenant_id, snapshot_month, drug_code)로 upsert.
· 월별 행수: 01=1070 · 02=1080 · 03=1087 · 04=1092 · 05=1103 (신규 등록 반영해 증가)
· monthly_snapshots에 snapshot_month 등 컬럼이 없으면 가산적 ALTER ADD(+롤백 SQL). 기존 데이터 보존.
※ [파일 2]의 이월+0(현재 시작 상태)와 [파일 3]의 과거 월마감(실제 이동값)은 별개다. 혼동 금지.

[파일 4] 보강_마약구분.csv  →  drugs.narcotic_type   (225행)
--------------------------------------------------------------------
narcotic_type='확인필요'만 보강. 일반 224 · 마약 1(IRCD5, 향정마약관리 원장).
규칙: 향정마약관리 목록=향정/마약, 그 외 확인필요=일반. 기존 향정/마약/한외마약 값은 불변.
※ DB(향정13·마약3·한외마약5) vs 원장(향정11·마약4)·한외마약 부재 차이는 덮어쓰기 전 확인.
