# CNC 약품관리 시스템 (cnc-pharmacy)

> 이 파일은 Claude Code가 매 작업 시작 시 자동으로 읽는 프로젝트 안내 메모입니다.
> 대화는 한국어로 진행해 주세요.

## 프로젝트 개요
- CNC재활의학과병원 약제과의 약품관리 웹 시스템입니다.
- 100~150병상 규모 재활/요양병원, 약사 1~2인 팀을 대상으로 한 SaaS 상용화를 목표로 발전시키는 중입니다.
- 배포 주소: cnc-pharmacy.netlify.app
- 저장소: ilovemeriel-netizen/cnc-pharmacy

## 기술 스택
- 프론트엔드: React + Vite
- 데이터베이스/인증: Supabase
- 배포/서버리스: Netlify (공공데이터 API는 Netlify Functions 프록시 경유)

## 자주 쓰는 명령어
- `npm run dev` — 개발 서버 실행 (작업 중 미리보기)
- `npm run build` — 배포용 빌드

## 주요 기능
- 월별 스냅샷, 리포트 탭, 대시보드 카드 네비게이션
- 슬라이드 편집 모달, 엑셀 일괄 등록, 정렬 가능한 표 헤더
- 공공데이터 API 프록시(Netlify Functions), CNC 병원 색상 시스템

## 도메인 규칙 (약품 관리)
- 약 600종 약품: 영양제 / 수액제 / 주사제 / 외용제 / 경구제 / 의약외품
- 로트(lot)별 유효기간 관리, 만료 3개월 전 알림
- 입고/출고/폐기/반품 연동, 단가 및 재고금액 추적
- 출고는 월 1회 일괄입력, 입고는 명세서 기준 수시 입력

## 작업 시 지켜야 할 규칙
- 코드 출력은 항상 처음부터 끝까지 전체로 제공 (중간 생략 금지)
- 파일을 수정하기 전에 무엇을 바꿀지 먼저 설명하고 확인받기

## 보안 (매우 중요)
- API 키, 비밀번호, 토큰을 코드나 이 파일에 절대 직접 적지 않는다.
- 비밀값은 환경변수로 분리: 로컬은 `.env`(반드시 `.gitignore`에 포함),
  배포는 Netlify 환경변수, Codespaces에서는 Codespaces Secrets 사용.
- 민감한 외부 API 호출은 Netlify Functions 프록시를 통해 처리한다.
